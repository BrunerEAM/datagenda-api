
BASE DE CONHECIMENTO SOBRE SEEDS (PARA DESENVOLVEDORES)

O que é um seed? 
No contexto de banco de dados, "seed" (do inglês semente) é um script que insere dados iniciais automaticamente.

Para que serve?

-Popular o banco com dados fictícios (ex: nomes de clientes, produtos etc)

-Testar funcionalidades (filtros, listagens, exportações)

-Evitar precisar cadastrar tudo manualmente após cada reset do banco

🛠 Como usar um seed no Knex?

Primeiramente ver o arquivo Knexfile.js para ver qual o dieretório do Seed.

1- Criar um arquivo seed: npx knex seed:make seed_clientes

2- Rodar o seed: npx knex seed:run

3- Se quiser rodar somente um seed específico: npx knex seed:run --specific=seed_clientes.js
